Welcome — Be smart with AI

This site teaches clear, practical steps for anyone who uses AI: how to ask good prompts, protect privacy, spot mistakes (hallucinations), verify results, and use AI ethically. No accounts, no downloads — just lessons you can read and share.

Who this is for

Students, parents, teachers, creators, and anyone who wants to use AI without getting tricked or sharing private data.

Lessons — Quick guides

1. Know the limits — AI can make confident mistakes
AI models don't "know" facts the way humans do. They predict text that looks right — sometimes it's wrong. Always verify important facts with reliable sources (official sites, academic articles, trusted news).
2. Protect your privacy — don't paste secrets
Never share personal IDs, bank details, passwords, or private messages in an AI chat. If you must process sensitive text, remove or anonymize it first. Check the service's privacy policy before uploading anything sensitive.
3. Ask for sources — request citations
When AI gives facts, ask: "Where did you get that?" or "Give sources." Then verify those sources independently. If the model can't provide verifiable sources, treat the answer as unverified.
4. Spot hallucinations — signs an answer is fake
Look for overly specific but unverifiable details, confident-sounding answers without sources, or invented quotes. Cross-check unusual claims with trusted sources.
5. Prompt safely — how to ask
Be clear and specific. Provide context but avoid private data. Use step-by-step prompts and ask the model to think step-by-step when you need careful reasoning.
6. Use AI responsibly — ethics and fairness
Don't use AI to deceive, impersonate, or produce harmful content. Credit AI when its output is used publicly. Consider bias: AI reflects the data it was trained on.

Publish & setup — no downloads required

Pick one of the methods below — all can be done in your browser and are free for simple sites.

Option A — GitHub Pages (recommended for control)

  1. Create a free GitHub account at github.com (web).
  2. Click New repository. Name it anything (or username.github.io for a user site).
  3. Open the repo, click Add file → Create new file. Name it index.html.
  4. Paste the entire content of this page (the code you copied from canvas) into that file and commit.
  5. Go to Settings → Pages (left sidebar). Under "Build and deployment" choose the branch (usually main) and folder / (root), then Save.
  6. Your site URL will be shown in Pages (example: https://username.github.io/repo-name/ or https://username.github.io/ for a repo named username.github.io).

Option B — Replit (very easy, instant preview)

  1. Create a free Replit account at replit.com.
  2. Click Create → select "HTML, CSS, JS" template.
  3. Open index.html in the editor, replace contents with this page, and click Run. Use the "Open in a new tab" or "Share" link to get a public URL.

Option C — Neocities (simple web host)

  1. Sign up at neocities.org.
  2. Create a new site, open the file editor, create index.html, paste the code, and save. Your site will be at yoursite.neocities.org.

Editing later

Edit the same index.html in the platform's web editor to change text, add lessons, or update links. Use commit/edit to publish updates.

FAQ

Q: Can I accept messages through this site?
A: You can link to email (mailto:) or use Google Forms. Without a server, the site itself cannot receive emails directly.

Q: Will AI answers be private?
A: Only if the service you use promises privacy. Avoid pasting sensitive data into third-party AIs.
